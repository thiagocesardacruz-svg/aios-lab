# QA Report Template

## Meta
- **ID:** qa-report-tmpl
- **Squad:** qa
- **Used by:** validate-deliverable, review-content-quality
- **Output format:** md
- **Version:** 1.0.0

## Variables
| Variable | Description | Required | Default |
|----------|-------------|----------|---------|
| `{{report_id}}` | Unique report identifier | yes | none |
| `{{artifact_name}}` | Name of reviewed artifact | yes | none |
| `{{artifact_type}}` | Type (code, content, design, etc.) | yes | none |
| `{{reviewed_by}}` | Reviewer agent ID | yes | none |
| `{{review_date}}` | Date of review | yes | none |
| `{{overall_status}}` | PASS, FAIL, or CONDITIONAL | yes | none |
| `{{checklist_used}}` | Checklist ID applied | yes | none |
| `{{findings_summary}}` | Summary of findings by severity | yes | none |
| `{{detailed_findings}}` | Detailed list of issues | no | "None" |
| `{{recommendations}}` | Recommended actions | no | "None" |

## Usage Notes
- Status is PASS only if all blocking items pass
- CONDITIONAL means pass with minor issues to fix
- Always include checklist reference for traceability
