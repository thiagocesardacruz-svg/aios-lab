## Pre-Requisitos

### Fundacao do Negocio
- [ ] Valor continuo claramente definido (o que cliente recebe todo mes)
- [ ] Oferta de atracao (trial/desafio) validada e funcionando
- [ ] Sistema de billing recorrente implementado
- [ ] Metricas de churn sendo rastreadas
- [ ] Metas de lucro de 30 dias definidas

### Inventario de Assets
- [ ] Lista completa de assets existentes (cursos, templates, ferramentas)
- [ ] Precos historicos reais documentados para cada asset
- [ ] Calculo do valor total do pacote de bonus
- [ ] Confirmacao: valor do bonus > primeiro pagamento de continuidade
- [ ] Verificacao: todos os bonus sao relacionados a oferta principal

## Tipo 1: Oferta de Bonus de Continuidade

### Design do Bonus
- [ ] **Bonus principal** definido com valor percebido > primeiro pagamento
  - Asset: _________________
  - Valor real: $_____________
  - Primeiro pagamento: $_____________
  - Ratio: ______x (deve ser >1x)

- [ ] **Bonus secundarios** listados com valores
  - Bonus 2: _________________ (valor: $______)
  - Bonus 3: _________________ (valor: $______)
  - Bonus 4: _________________ (valor: $______)
  - Valor total empilhado: $_____________

- [ ] **Tipo de bonus** identificado
  - [ ] Mais do mesmo (extensao do que compram)
  - [ ] Complementar (adiciona ao produto principal)
  - [ ] Upgrade (versao superior)
  - [ ] Titulo/Status (Silver, Gold, Diamond)

- [ ] **Cross-bonus** considerado (se aplicavel)
  - [ ] Digital + Fisico (se oferta digital)
  - [ ] Fisico + Digital (se oferta fisica)

### Regras de Bonus Verificadas
- [ ] Precos de bonus sao REAIS (nao inventados)
- [ ] Bonus e RELACIONADO a oferta principal
- [ ] Bonus usa assets EXISTENTES (custo marginal zero preferido)
- [ ] Ancora de valor e CRIVEL para o cliente

### Script de Apresentacao
- [ ] Script criado: Vender BONUS primeiro (nao membership)
- [ ] Script criado: Perguntar "Quer saber como conseguir gratis?"
- [ ] Script criado: Revelar que membership da bonus
- [ ] Script criado: Empilhar bonus com valores individuais
- [ ] Script criado: Oferecer opcao standalone

## Tipo 3: Oferta de Taxa Dispensada

### Estrutura da Taxa
- [ ] **Preco mensal** definido: $_____________
- [ ] **Taxa de configuracao** definida: $_____________
  - Range recomendado: 3-5x mensal
  - Sua taxa: ______x mensal
  - [ ] Taxa >= 3x mensal
  - [ ] Taxa <= 5x mensal

- [ ] **Duracao do compromisso** definida: _____ meses
  - [ ] Minimo 12 meses (requerido)

### Opcoes para Cliente
- [ ] **Opcao A** (Mes a Mes) documentada:
  - Taxa: $_____________
  - Primeiro mes: $_____________
  - Total hoje: $_____________
  - Mensalidade: $_____________
  - Cancelamento: A qualquer momento

- [ ] **Opcao B** (Compromisso) documentada:
  - Taxa: Dispensada
  - Compromisso: _____ meses
  - Mensalidade: $_____________
  - Se cancelar antes: Pagar taxa de $_____________

### Documentacao Legal
- [ ] Documento para cliente rubricar criado
- [ ] Linguagem clara sobre taxa se cancelar cedo
- [ ] Processo de cobranca de taxa definido

### Regras Verificadas
- [ ] Taxa desaparece APOS completar compromisso
- [ ] Mecanismo de investigacao se >5% cancelam cedo
- [ ] Servico e de resultado de LONGO PRAZO (SEO, investing, weight loss)

### Tatica Avancada (Opcional)
- [ ] "Taxa para causa odiada" considerada
  - Script criado: "Qual causa voce odeia?"
  - Mecanismo de doacao definido

## Troubleshooting Quick Reference

| Problema | Causa Provavel | Solucao |
|----------|----------------|---------|
| Muitos escolhem standalone | Multiplicador baixo | Aumentar multiplicador |
| Muitos mes-a-mes (waived fee) | Taxa baixa | Aumentar taxa (3-5x) |
| >5% cancelam antes do compromisso | Produto ruim | Investigar produto, NAO aumentar taxa |
| Baixo uptake de prepayment | Nao oferecendo | Sempre oferecer apos aceitar continuidade |
| Bonus nao ressoa | Nao relacionado | Trocar por bonus relevante |
