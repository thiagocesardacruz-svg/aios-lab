## Pre-Flight: Pré-Requisitos

### Validação de Prontidão
- [ ] **Produto validado:** Pelo menos 10 clientes satisfeitos pagantes
- [ ] **CAC conhecido:** Custo de aquisição documentado e consistente
- [ ] **Capacidade de entrega:** Consegue entregar 2x volume atual
- [ ] **Sistema de cobrança:** Processador aceita planos e recorrência
- [ ] **Mentalidade alinhada:** "Todo não é rejeição, é redirecionamento"
- [ ] **Caixa mínimo:** 3 meses de despesas OU linha de crédito disponível
- [ ] **Tracking instalado:** Métricas de conversão por estágio configuradas

### Red Flags (NÃO está pronto se)
- [ ] Taxa de reembolso > 5%
- [ ] Não consegue explicar valor em 30 segundos
- [ ] Não tem depoimentos/casos de sucesso
- [ ] Margem bruta < 30%
- [ ] Churn > 10% mensal

**Se qualquer red flag marcado:** PARAR e resolver antes de prosseguir.

## ESTÁGIO 1: Ofertas de Atração (Get Money)

### 1.1 Seleção de Tipo de Oferta
- [ ] Identificar tipo de negócio (serviço / local / físico / digital)
- [ ] Mapear característica principal do produto
- [ ] Selecionar tipo de oferta de atração mais adequado:
  - [ ] Ganhe Seu Dinheiro de Volta (atividades de começar/parar)
  - [ ] Sorteios (geração massiva de leads)
  - [ ] Oferta Isca (maximizar conversão geral)
  - [ ] Compre X, Ganhe Y Grátis (produtos físicos)
  - [ ] Pague Menos Agora (workshops, trials)

### 1.2 Configuração: Ganhe Seu Dinheiro de Volta
- [ ] Definir resultado mensurável do programa
- [ ] Criar 3-5 critérios de qualificação:
  - [ ] Critério é fácil de rastrear
  - [ ] Critério gera resultados aos clientes
  - [ ] Critério inclui divulgação do negócio
- [ ] Definir como aplicar crédito (100% próximo programa OU espalhado)
- [ ] Documentar script de apresentação
- [ ] Preparar materiais de comunicação

[Fonte: Seção 2, "Como Criar Seus Critérios"]

### 1.3 Configuração: Sorteios
- [ ] Definir grande prêmio (= produto principal com valor âncora)
- [ ] Criar oferta promocional (desconto 10-30% margens)
- [ ] Estabelecer prazo (3-7 dias)
- [ ] Preparar processo de coleta de informações
- [ ] Criar script para "bolsa parcial"
- [ ] Consultar assessoria jurídica sobre regulamentação

[Fonte: Seção 2, "Sorteios"]

### 1.4 Configuração: Oferta Isca
- [ ] Criar versão ISCA (básica, sem suporte, sem garantia)
- [ ] Criar versão PREMIUM (completa, personalizada, garantida)
- [ ] Garantir contraste ENORME entre as duas
- [ ] Preparar script de permissão ("coisas grátis ou resultados?")
- [ ] Treinar apresentação com entusiasmo para premium

[Fonte: Seção 2, "Oferta Isca"]

### 1.5 Configuração: Compre X, Ganhe Y Grátis
- [ ] Calcular preço unitário normal
- [ ] Definir multiplicador (mínimo 3x para "2 grátis")
- [ ] Garantir que "grátis" > "pago" em quantidade
- [ ] Ajustar estoque/capacidade para entrega
- [ ] Criar urgência (tempo ou quantidade limitada)

[Fonte: Seção 2, "Compre X, Ganhe Y Grátis"]

### 1.6 Configuração: Pague Menos Agora
- [ ] Definir preço "depois" (full price)
- [ ] Definir preço "agora" (20-50% desconto)
- [ ] Listar bônus exclusivos para "pagar agora"
- [ ] Garantir resultado claro sim/não
- [ ] Preparar processo de coleta de cartão

[Fonte: Seção 2, "Pague Menos Agora"]

### 1.7 Validação Estágio 1
- [ ] **Lucro 30 dias >= CAC**
- [ ] **Conversão >= 10%**
- [ ] **Testado com >= 10 clientes**
- [ ] **Scripts documentados**

## ESTÁGIO 2: Ofertas de Upsell (More Money)

### 2.1 Mapeamento Problema-Solução
- [ ] Listar solução principal oferecida
- [ ] Identificar 10 problemas que a solução CRIA
- [ ] Para cada problema, criar oferta de upsell
- [ ] Ordenar upsells por margem de lucro (maior primeiro)
- [ ] Validar timing de cada upsell na jornada

[Fonte: Seção 3, "Exercício 1: Mapeamento Problema-Solução"]

### 2.2 Upsell Clássico
- [ ] Identificar problema IMEDIATO após compra principal
- [ ] Estruturar frase "Você não pode ter X sem Y"
- [ ] Criar script completo:
  - [ ] Conectar solução A ao problema B
  - [ ] Mencionar "90% dos clientes também pegam"
  - [ ] Oferecer preço especial vs normal
  - [ ] Incluir "Você não quer mais nada, quer?"
- [ ] Treinar tom casual, não ansioso
- [ ] Configurar BAMFAM (agendar próximo encontro)

[Fonte: Seção 3, "O Upsell Clássico"]

### 2.3 Menu Upsell (4 Táticas)
- [ ] **DESVENDA:**
  - [ ] Listar produtos/serviços disponíveis
  - [ ] Identificar quais NÃO se aplicam ao cliente
  - [ ] Criar script de "riscar" itens irrelevantes
- [ ] **PRESCREVE:**
  - [ ] Para itens que se aplicam, detalhar:
    - [ ] Quantidade exata
    - [ ] Quando usar
    - [ ] Como usar
    - [ ] Por que usar
- [ ] **AB:**
  - [ ] Para cada item, criar variações A e B
  - [ ] Preparar "empurrãozinho" ("Este é meu favorito")
- [ ] **CARTÃO EM ARQUIVO:**
  - [ ] Configurar sistema de cartão salvo
  - [ ] Preparar frase "Quer usar o cartão em arquivo?"

[Fonte: Seção 3, "Menu Upsells"]

### 2.4 Anchor Upsell
- [ ] Criar oferta PREMIUM (5-10x preço principal)
- [ ] Garantir que premium é REAL (não falsa)
- [ ] Definir features primárias (mesmas nas duas)
- [ ] Definir features secundárias (premium only)
- [ ] Preparar processo de 5 passos:
  - [ ] Apresentar âncora
  - [ ] Esperar engasgo
  - [ ] Vir ao resgate ("Você se importa com...?")
  - [ ] Apresentar principal
  - [ ] Pedir pagamento

[Fonte: Seção 3, "Anchor Upsell"]

### 2.5 Rollover Upsell
- [ ] Identificar situações de uso:
  - [ ] Reengajar clientes antigos (6+ meses)
  - [ ] Resgatar clientes chateados
  - [ ] Roubar clientes de concorrentes
  - [ ] Upsell clientes atuais
- [ ] Calcular valor máximo de crédito (≤25% da nova oferta)
- [ ] Definir como aplicar (antecipado ou espalhado)
- [ ] Criar urgência (oferta única)

[Fonte: Seção 3, "Rollover Upsell"]

### 2.6 Validação Estágio 2
- [ ] **Aceitação upsell >= 30%**
- [ ] **LTV 30 dias >= 2x compra inicial**
- [ ] **>= 3 upsells mapeados e funcionando**
- [ ] **Scripts de cada tipo documentados**

## ESTÁGIO 3: Ofertas de Downsell (Recuperar)

### 3.1 Regras do Downselling (CRÍTICO)
- [ ] Internalizar: "Não = não para ESTA oferta, não todas"
- [ ] NUNCA baixar preço da mesma coisa
- [ ] Sempre trocar: dar algo = receber algo
- [ ] Personalizar, não pressionar
- [ ] 100 formas de oferecer > 100 produtos

[Fonte: Seção 4, "As regras do downselling"]

### 3.2 Downsell de Plano de Pagamento (7 Passos)
- [ ] **Passo 1:** Recompensar pré-pagamento
  - [ ] Apresentar preço com juros inclusos
  - [ ] Oferecer desconto se pré-pagar
- [ ] **Passo 2:** Financiamento externo
  - [ ] Oferecer financiamento de terceiros
  - [ ] Script: "Você prefere decidir seus termos?"
  - [ ] Opção de layaway
- [ ] **Passo 3:** Metade agora, metade depois
  - [ ] Perguntar quando recebe
  - [ ] Alinhar com salário
- [ ] **Passo 4:** Verificar desejo (escala 1-10)
  - [ ] Se >= 8: continuar downselling
  - [ ] Se <= 7: vender algo diferente
- [ ] **Passo 5:** Três pagamentos
  - [ ] 1/3 agora + 1/3 + 1/3
- [ ] **Passo 6:** Pagamentos uniformes
  - [ ] Espalhar por duração do serviço
- [ ] **Passo 7:** Trial gratuito (último recurso)

[Fonte: Seção 4, "Downsells de Plano de Pagamento"]

### 3.3 Downsell Gangorra (Versão Simplificada)
- [ ] Preparar script:
  - [ ] "Pagamentos gigantes ou pequenos?"
  - [ ] "Se pré-pagar, zero pagamentos mensais"
  - [ ] "Quanto mais der agora, menores os pagamentos"
- [ ] Treinar posicionamento lado a lado com cliente

[Fonte: Seção 4, "Downselling gangorra"]

### 3.4 Trial com Penalidade
- [ ] Definir termos para evitar taxa:
  - [ ] Ações que fazem excelentes clientes
  - [ ] Check-ins obrigatórios (oportunidades de upsell)
- [ ] Decidir estrutura de taxa:
  - [ ] Taxa única grande OU
  - [ ] Múltiplas taxas pequenas
- [ ] Preparar processo de 5 passos:
  - [ ] Oferecer trial por ÚLTIMO
  - [ ] SEMPRE pegar cartão
  - [ ] Vender ficar e pagar
  - [ ] Explicar taxas APÓS cartão
  - [ ] Tornar check-ins obrigatórios
- [ ] Script: "É assim que sempre fizemos"

[Fonte: Seção 4, "Trial com Penalidade"]

### 3.5 Downsell de Recursos
- [ ] Mapear recursos removíveis:
  - [ ] Quantidade (menos sessões, menos tempo)
  - [ ] Qualidade (materiais, suporte, velocidade)
  - [ ] Recursos inteiros (garantia, chamadas)
  - [ ] DIY vs DFY
- [ ] Para cada redução, calcular novo preço
- [ ] Ordenar do maior para menor valor (hack da garantia)
- [ ] Script: "Tire algo, baixe preço, pergunte: E agora?"

[Fonte: Seção 4, "Downsells de Recursos"]

### 3.6 Validação Estágio 3
- [ ] **Recovery rate >= 40%**
- [ ] **Conversão total >= 50%**
- [ ] **Taxa de reembolso <= 5%**
- [ ] **Todos os scripts documentados**

## ESTÁGIO 4: Ofertas de Continuidade (Max Money)

### 4.1 Bônus de Continuidade
- [ ] Criar bônus ÚNICO de alto valor:
  - [ ] Valor > primeiro pagamento
  - [ ] Relacionado à oferta principal
  - [ ] De preferência algo que já existe
- [ ] Criar bônus MENSAIS exclusivos para membros
- [ ] Preparar ancoragem de valor:
  - [ ] Vender benefícios do bônus primeiro
  - [ ] Revelar como conseguir grátis (se juntar)
  - [ ] Empilhar bônus adicionais

[Fonte: Seção 5, "Ofertas de Bônus de Continuidade"]

### 4.2 Precificação Continuidade vs Independente
- [ ] Definir preço mensal de continuidade
- [ ] Calcular preço independente baseado no % desejado:
  - [ ] 50% continuidade → 1.33x
  - [ ] 60% continuidade → 1.66x
  - [ ] 70% continuidade → 2.00x
  - [ ] 80% continuidade → 2.33x
  - [ ] 90% continuidade → 2.66x
- [ ] Testar e ajustar baseado em resultados reais

[Fonte: Seção 5, "Precificação para continuidade"]

### 4.3 Upsell de Pré-Pagamento em Massa
- [ ] Criar oferta "Compre X meses, ganhe Y grátis"
- [ ] Calcular breakeven (1 em 8 = +50% lucro 30 dias)
- [ ] Oferecer imediatamente após aceitar continuidade
- [ ] Aplicar urgência (apenas agora)

[Fonte: Seção 5, "Se você quer ainda mais dinheiro"]

### 4.4 Oferta de Taxa Dispensada
- [ ] Definir taxa de configuração (3-5x mensalidade)
- [ ] Estruturar duas opções:
  - [ ] Opção A: Mês a mês + taxa (cancele quando quiser)
  - [ ] Opção B: Compromisso X meses, taxa dispensada
- [ ] Definir duração do compromisso (mínimo 12 meses)
- [ ] Estabelecer que taxa é paga se quebrar compromisso
- [ ] Eliminar taxa após cliente cumprir compromisso
- [ ] [OPCIONAL] Taxa vai para causa que cliente odeia

[Fonte: Seção 5, "Oferta de Taxa Dispensada"]

### 4.5 Otimização de Churn
- [ ] Analisar frequência de cobrança atual
- [ ] Considerar migração para frequência menor:
  - [ ] Mensal → Trimestral: -50% churn
  - [ ] Trimestral → Anual: -60% churn
- [ ] Criar incentivos para períodos mais longos
- [ ] Implementar prevenção de churn proativa

[Fonte: Seção 4, "ProfitWell - dados de 14.000 empresas"]

### 4.6 Validação Estágio 4
- [ ] **>= 60% em continuidade**
- [ ] **Churn mensal <= 5%**
- [ ] **Valor bônus >= 1.5x primeiro pagamento**
- [ ] **Estrutura de taxa documentada**

## Integração: Modelo Completo

### 5.1 Montagem do Modelo
- [ ] Documentar fluxo completo:
  ```
  LEAD → ATRAÇÃO → UPSELL(s) → DOWNSELL(s) → CONTINUIDADE
  ```
- [ ] Mapear pontos de decisão
- [ ] Identificar métricas por estágio
- [ ] Criar dashboard de acompanhamento

### 5.2 Cálculo de Lucro de 30 Dias
- [ ] Calcular usando fórmula:
  ```
  L30 = (P1 × C1) + Σ(Un × Cn × Vn) - CAC - COGS
  ```
- [ ] Validar L30 > 0
- [ ] Projetar L30 com otimizações

[Fonte: Framework Completo, "Fórmula do Lucro de 30 Dias"]

### 5.3 Validação Final
- [ ] **Lucro 30 dias >= 1x CAC** (mínimo)
- [ ] **Lucro 30 dias >= 2x CAC** (bom)
- [ ] **Lucro 30 dias >= 3x CAC** (excelente)
- [ ] **LTV 30 dias >= 3x CAC**
- [ ] **Capacidade operacional >= 2x atual**
- [ ] **Todos scripts e processos documentados**

## Plano de Implementação 30-60-90 Dias

### Primeiros 30 Dias: FUNDAÇÃO (Estágio 1)

**Semana 1: Descoberta e Mapeamento**
- [ ] Mapear jornada atual do cliente (cada touchpoint)
- [ ] Identificar 10 problemas criados por sua solução
- [ ] Escolher tipo de oferta de atração

**Semana 2: Teste de Atração**
- [ ] Criar materiais da oferta
- [ ] Testar com 10 prospects
- [ ] Documentar script vencedor

**Semana 3: Primeiro Upsell**
- [ ] Criar upsell clássico "não pode ter X sem Y"
- [ ] Testar com todos que compraram oferta principal
- [ ] Medir taxa de aceitação (meta: 30%)

**Semana 4: Downsell Básico**
- [ ] Implementar plano de pagamento simples
- [ ] Oferecer para todos os "nãos"
- [ ] Calcular lucro de 30 dias total

**Checkpoint Semana 4:**
- [ ] Lucro 30 dias >= $0
- [ ] Script de atração validado
- [ ] Pelo menos 1 upsell funcionando

### 30-60 Dias: ACELERAÇÃO (Estágio 2)

**Semana 5-6: Menu de Ofertas**
- [ ] Criar 3 níveis de serviço (mínimo, padrão, premium)
- [ ] Implementar "unselling" para personalização

**Semana 7-8: Otimização de Conversão**
- [ ] Implementar anchor com premium 5x preço
- [ ] Criar trial com penalidade para hesitantes
- [ ] Configurar rollover para clientes antigos

**Checkpoint Semana 8:**
- [ ] Lucro 30 dias >= 1x CAC
- [ ] Aceitação upsell >= 30%

### 60-90 Dias: MAESTRIA (Estágios 3-4)

**Semana 9-10: Continuidade Dominante**

## Sinais de Sucesso vs Problema

### Sinais de Que Está Funcionando

**Primeiras 24h:**
- [ ] Primeiro "sim" com nova técnica
- [ ] Cliente aliviado com downsell
- [ ] Upsell aceito "naturalmente"

**Primeira Semana:**
- [ ] Taxa de conversão +20%
- [ ] Ticket médio +40%
- [ ] Equipe animada com resultados

**Primeiro Mês:**
- [ ] Lucro 30 dias positivo
- [ ] CAC < 30% do LTV 30 dias
- [ ] Fila de espera para serviço

[Fonte: Framework Completo, "Sinais de Que Está Funcionando"]

### Sinais de Que Precisa Ajustar

**Red Flags:**
- [ ] Conversão inicial < 10% → Oferta errada
- [ ] Ninguém aceita upsell → Timing/relevância
- [ ] Alto reembolso > 5% → Expectativas desalinhadas
- [ ] Churn > 10% mensal → Produto não entrega

**Troubleshooting:**
- SE baixa conversão → ENTÃO mude de ganhe dinheiro para trial
- SE não aceitam upsell → ENTÃO mova para momento diferente
- SE muito reembolso → ENTÃO ajuste critérios e promessas
- SE alto churn → ENTÃO adicione onboarding e quick wins

[Fonte: Framework Completo, "Sinais de Que Precisa Ajustar"]

### 5 Ofertas de Atração
1. Ganhe Seu Dinheiro de Volta
2. Sorteios
3. Oferta Isca
4. Compre X, Ganhe Y Grátis
5. Pague Menos Agora ou Mais Depois

### 4 Ofertas de Upsell
1. Upsell Clássico ("Você não pode ter X sem Y")
2. Menu Upsell (Desvenda + Prescreve + AB + Cartão)

1. Plano de Pagamento (7 passos)
2. Trial com Penalidade

1. Bônus de Continuidade
2. Desconto de Continuidade (precificação 1.33x-2.66x)
3. Taxa Dispensada (3-5x mensalidade)

### Benchmarks Chave
- CAC coberto em 30 dias
- Conversão fria: 10-25%
- Aceitação upsell: 30-50%
- Recovery downsell: 40-60%
- Continuidade: 60-80%
- Churn mensal: < 5%
- Lucro 30 dias: >= 2x CAC (bom)
