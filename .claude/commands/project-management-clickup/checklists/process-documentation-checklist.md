# Checklist: Process Documentation

> Checklist para documentação de processos no ClickUp

---

## Pré-Documentação

### Preparação
- [ ] Identificar o processo a ser documentado
- [ ] Definir o owner do processo
- [ ] Agendar entrevistas com stakeholders
- [ ] Coletar documentação existente (se houver)
- [ ] Definir escopo (início e fim do processo)

### Stakeholders
- [ ] Listar todas as pessoas envolvidas
- [ ] Identificar quem executa cada etapa
- [ ] Mapear handoffs entre pessoas/times
- [ ] Confirmar disponibilidade para validação

---

## Durante a Documentação

### Mapeamento
- [ ] Documentar trigger (o que inicia o processo)
- [ ] Listar todas as etapas em ordem
- [ ] Identificar decisões/bifurcações
- [ ] Documentar inputs de cada etapa
- [ ] Documentar outputs de cada etapa
- [ ] Mapear ferramentas utilizadas
- [ ] Identificar SLAs esperados

### Detalhamento
- [ ] Descrever "como fazer" cada etapa
- [ ] Incluir screenshots quando necessário
- [ ] Documentar exceções e edge cases
- [ ] Listar erros comuns e como resolver
- [ ] Incluir templates utilizados
- [ ] Vincular documentos de referência

### Fluxograma
- [ ] Criar representação visual do processo
- [ ] Incluir decisões com diamantes
- [ ] Mostrar responsáveis em cada etapa
- [ ] Validar fluxo com executores

---

## Pós-Documentação

### Validação
- [ ] Revisar com quem executa o processo
- [ ] Validar com gestor/owner
- [ ] Testar documentação com pessoa nova
- [ ] Incorporar feedback

### Publicação
- [ ] Criar documento no ClickUp Docs
- [ ] Configurar permissões de acesso
- [ ] Vincular a tarefas relacionadas
- [ ] Adicionar à base de conhecimento
- [ ] Comunicar disponibilidade ao time

### Manutenção
- [ ] Definir responsável por atualizações
- [ ] Agendar revisão periódica (trimestral)
- [ ] Criar processo para sugestões de melhoria
- [ ] Versionar alterações significativas

---

## Critérios de Qualidade

### O documento DEVE ter:
- [ ] Título claro e descritivo
- [ ] Data de criação e última atualização
- [ ] Owner definido
- [ ] Objetivo do processo
- [ ] Pré-requisitos para execução
- [ ] Passo a passo detalhado
- [ ] Resultados esperados
- [ ] Contato para dúvidas

### O documento PODE ter (recomendado):
- [ ] Fluxograma visual
- [ ] Screenshots ilustrativos
- [ ] Templates prontos para uso
- [ ] FAQ com dúvidas comuns
- [ ] Histórico de versões
- [ ] Links para processos relacionados

---

## Métricas de Sucesso

| Métrica | Target |
|---------|--------|
| Tempo de onboarding reduzido | -30% |
| Erros de execução | -50% |
| Perguntas repetitivas | -70% |
| Satisfação do time | > 4/5 |
