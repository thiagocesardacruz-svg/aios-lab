## Part 1: Purpose

Create high-converting Video Sales Letter (VSL) scripts that guide prospects through a complete psychological journey from attention to action. VSLs control viewer exposure time and deliver persuasive messaging in a format that maximizes conversions.

**Core Philosophy:**
```
"VSL controls o tempo de exposicao a mensagem, maximizando persuasao.
A person who watches a VSL cannot skim. They experience your pitch
at YOUR pace, in YOUR sequence, with YOUR emphasis."
```

**VSL Role in Funnel:**
```
Registration Page → Thank You Page (VSL) → Upsell Page (VSL) → OTO Page (VSL)
                        ↓                      ↓                    ↓
                 VIP Upgrade            Book Bundle          High-Ticket Offer
```

**Business Impact:**
- VSL watch time: >50% completion rate
- VSL-to-CTA click: >15% on warm traffic
- VSL page conversion: >5% for front-end offers
- OTO acceptance rate: >20% on backend offers

## Part 2: Scope

### When This SOP Applies
- Creating video sales letter scripts for product launches
- Building thank you page VSLs for immediate upsells
- Designing OTO (One-Time Offer) sequences
- Writing recap VSLs for event follow-up
- Creating VIP upgrade presentations
- Building book/course launch scripts

### When This SOP Does NOT Apply
- Short-form ads under 60 seconds (use hooks-ads SOP)
- Webinar presentations over 60 minutes (use webinar SOP)

## Part 4: Core Framework - The VSL Architecture

### Framework Overview

```
+===========================================================================+
|                    VSL SCRIPT ARCHITECTURE                                 |
+===========================+================================================+
| PHASE 1: HOOK             | First 15-30 seconds - Stop the scroll          |
| (Attention Capture)       | Pattern interrupt + Curiosity gap              |
+---------------------------+------------------------------------------------+
| PHASE 2: CONTEXT          | 30 sec - 2 min - Establish why they should     |
| (Background + Authority)  | listen to YOU about THIS topic                 |
+---------------------------+------------------------------------------------+
| PHASE 3: PROBLEM AGITATE  | 2-5 min - Make them FEEL the pain              |
| (Pain Amplification)      | Specific problems + Emotional resonance        |
+---------------------------+------------------------------------------------+
| PHASE 4: SOLUTION REVEAL  | 2-5 min - Your unique mechanism                |
| (The "What")              | What it is + Why it works + How it's different |
+---------------------------+------------------------------------------------+
| PHASE 5: PROOF STACK      | 2-5 min - Evidence it works                    |
| (Belief Building)         | Data + testimonials + logic + credentials      |
+---------------------------+------------------------------------------------+
| PHASE 6: OFFER STACK      | 2-3 min - What they get                        |
| (Value Construction)      | Components + bonuses + value assignment        |
+---------------------------+------------------------------------------------+
| PHASE 7: CLOSE            | 1-3 min - Call to action + urgency             |
| (Decision + Action)       | Price reveal + guarantee + CTA + deadline      |
+---------------------------+------------------------------------------------+
```

## Part 6: Decision Rules

### DR-001: VSL Duration by Traffic Temperature
```yaml
rule: VSL_DURATION_MATCH
trigger: Planning VSL
condition:
  - IF traffic == cold THEN duration(8-15min)
  - IF traffic == warm THEN duration(3-8min)
  - IF traffic == hot (just purchased) THEN duration(3-5min)
action: Match VSL length to how much trust you've built
confidence: HIGH
```

### DR-002: Proof Depth Selection
```yaml
rule: PROOF_DEPTH
trigger: Building proof section
condition:
  - IF offer_price < $100 THEN basic_proof(2-3 testimonials)
  - IF offer_price $100-$1000 THEN medium_proof(3-5 testimonials + data)
  - IF offer_price > $1000 THEN heavy_proof(5+ testimonials + extensive data + credentials)
action: Scale proof to match perceived risk
confidence: HIGH
```

### DR-003: Urgency Type Selection
```yaml
rule: URGENCY_TYPE
trigger: Planning close
condition:
  - IF launch_based THEN deadline_urgency("expires Monday midnight")
  - IF capacity_limited THEN spot_urgency("limited to X seats")
  - IF bonus_limited THEN bonus_urgency("bonus disappears at X")
  - IF no_real_scarcity THEN SKIP or soft_urgency("cost of waiting")
action: Only use urgency you can actually enforce
confidence: HIGH
```

### DR-004: OTO Sequencing
```yaml
rule: OTO_SEQUENCE
trigger: Post-purchase funnel
condition:
  - IF just_purchased_low_ticket THEN offer_mid_ticket_OTO
  - IF just_purchased_mid_ticket THEN offer_high_ticket_OTO
  - IF declined_OTO THEN offer_downsell_OTO (30-50% lower)
  - IF accepted_OTO THEN additional_OTO or thank_you_only
action: Ascend or downsell based on response
confidence: HIGH
```

### DR-005: Problem Agitation Depth
```yaml
rule: PROBLEM_DEPTH
trigger: Writing problem section
condition:
  - IF audience_awareness == problem_aware THEN light_agitation()
  - IF audience_awareness == unaware THEN heavy_agitation()
  - IF solution_focused THEN transition_quickly()
action: Only agitate problems they haven't fully processed
confidence: MEDIUM
```

## Part 7: Exceptions & Edge Cases

### Exception 1: Thank You VSL After High-Ticket Sale
**Trigger:** Post $5K+ purchase
**Action:**
- Minimize additional selling
- Focus on confirmation and onboarding
- If OTO exists, make it clearly optional
- Emphasize implementation over more buying

### Exception 2: Short Attention Span Audience
**Trigger:** Younger demo or social media traffic
**Action:**
- Cut VSL duration by 30-50%
- Front-load the hook
- Use faster pacing
- Consider text overlays for key points

### Exception 3: Returning Customer OTO
**Trigger:** They've bought from you before
**Action:**
- Skip credibility section
- Acknowledge relationship
- Reference past purchase success
- Faster to offer

### Exception 4: Pre-Event Registration VSL
**Trigger:** Trying to get event signups
**Action:**
- Keep under 3 minutes
- Focus on event value, not product
- Tease what they'll learn
- Make registration frictionless

### Exception 5: International Audience
**Trigger:** Non-native speakers
**Action:**
- Slower pace
- Clearer enunciation
- Less idiom/slang
- Consider subtitles
- Simpler sentence structure
