## Pre-Flight Checklist

Before starting landing page creation, verify:

### Offer Definition
- [ ] Single, clear offer defined (one page = one offer = one decision)
- [ ] Target avatar deeply understood (language, pains, desires)
- [ ] Value proposition articulated
- [ ] Price point determined
- [ ] Guarantee terms approved

### Assets Ready
- [ ] Professional creator photo available
- [ ] 2-3 quantifiable credentials documented
- [ ] Testimonials collected (3-5 with numerical results)
- [ ] Logo assets prepared
- [ ] VSL recorded (8-15 min cold, 15-45 min warm) OR hero image ready

### Technical Infrastructure
- [ ] Hosting configured (uptime >99.9%)
- [ ] SSL certificate installed
- [ ] Payment processor integrated
- [ ] CDN enabled
- [ ] Daily backup configured

## Phase 1: Above the Fold (0-3 Segundos)

### 1.1 Headline Magnetica
- [ ] Uses structure: "Como [RESULTADO ESPECIFICO] em [TIMEFRAME] sem [DOR PRIMARIA]"
- [ ] First thing visible on page
- [ ] Largest font on page
- [ ] Passes read aloud test (sounds natural)
- [ ] Makes visitor think "como assim?" or "isso e possivel?"

### 1.2 Subheadline de Credibilidade
- [ ] Uses formula: "O [MECANISMO] usado por [NUMERO] [AVATAR] para [RESULTADO COMPROVADO]"
- [ ] Positioned directly below headline
- [ ] Font 60% smaller than headline
- [ ] Increases Perceived Likelihood with specificity
- [ ] Does NOT sound like sales jargon (Red Flag check)

### 1.3 CTA Principal
- [ ] Text is benefit-oriented ("Quero 30 Clientes em 90 Dias")
- [ ] NOT action-oriented ("Comprar", "Inscrever")
- [ ] Minimum 200px width on mobile
- [ ] Contrast ratio 3:1 minimum
- [ ] Color that "screams" (stands out)
- [ ] Includes arrow (→) for 12-26% CTR increase
- [ ] Visible without scrolling
- [ ] Within above-the-fold area

### 1.4 Prova Social Imediata
- [ ] Uses format: "⭐⭐⭐⭐⭐ X+ Alunos | Y% Satisfacao | Resultados em Z dias"
- [ ] Positioned below CTA or integrated in header
- [ ] Uses REAL numbers (never estimates)
- [ ] Creates instant trust before first objection

### 1.5 Indicador de Urgencia
- [ ] Type is one of: timer regressivo, vagas limitadas, bonus expirando
- [ ] **CRITICAL:** Urgency is 100% REAL (fake scarcity destroys trust)
- [ ] Design: visible but not aggressive
- [ ] Color: red or orange
- [ ] Position: near CTA or fixed bar at top

---

## Phase 2: Experiencia Movel (60%+ do Trafego)

### 2.1 Velocidade de Carregamento
- [ ] Load time < 3 seconds on 3G mobile connection
- [ ] Google PageSpeed Insights score > 90
- [ ] Images compressed
- [ ] CDN configured
- [ ] Lazy loading implemented
- [ ] Tested with real mobile data (not just simulators)

### 2.2 Layout Responsivo
- [ ] Text legible without zoom (minimum 16px)
- [ ] Buttons minimum 44px height (finger-friendly)
- [ ] Elements not "stuck together"
- [ ] Easy thumb navigation
- [ ] Tested on real devices (smartphone, tablet, desktop)

### 2.3 Formularios Otimizados
- [ ] Minimum fields necessary (name + email for lead magnet)
- [ ] Correct input types (email, phone, number)
- [ ] Autocomplete enabled for all fields
- [ ] Real-time validation (not just on submit)

---

## Phase 3: Estrutura de Persuasao

### 3.1 Hierarquia Visual Clara
- [ ] Flow follows: Headline → Subheadline → VSL/Imagem → Beneficios → CTA
- [ ] Font sizes: headline largest, then subheadline, then body
- [ ] Important elements have visual contrast
- [ ] White space between sections (breathing room)

### 3.2 VSL ou Imagem Hero
**If VSL:**
- [ ] Duration: 8-15 min for cold traffic
- [ ] Duration: 15-45 min for warm traffic
- [ ] Thumbnail has obvious play button
- [ ] Thumbnail has benefit text

**If Image:**
- [ ] Mockup shows result/transformation
- [ ] Centered positioning
- [ ] 16:9 aspect ratio
- [ ] Responsive design

### 3.3 Bullets de Beneficios (NAO Features)
- [ ] Uses formula: "✓ [RESULTADO] para que voce possa [BENEFICIO EMOCIONAL]"
- [ ] Quantity: 5-7 bullets maximum (more dilutes impact)
- [ ] Each bullet makes prospect think "preciso disso"
- [ ] Design: green checkmarks
- [ ] Design: generous spacing
- [ ] NO features without emotional benefits

**Anti-Pattern Check:**
- [ ] Not saying "10 modulos + 50 templates + grupo no Facebook"
- [ ] For each feature listed, "...para que voce possa [RESULTADO]" is complete

---

## Phase 4: Qualificacao e Objecoes

### 4.1 Secao "Para Quem E"
- [ ] Uses structure: "Este [PRODUTO] e perfeito se: ☑️ [SITUACAO] ☑️ [DOR] ☑️ [DESEJO]"
- [ ] Position: after presenting solution, before price
- [ ] Function: increases Perceived Likelihood ("foi feito para mim")
- [ ] Tone: specific and inclusive

### 4.2 Secao "Para Quem NAO E"
- [ ] Uses structure: "ATENCAO: Isso NAO e para voce se: ❌ [DESQUALIFICADOR]"
- [ ] Position: immediately after "Para Quem E"
- [ ] Function: exclusivity increases desire + qualifies better
- [ ] Tone: firm but respectful

### 4.3 FAQ das 5 Maiores Objecoes
- [ ] Addresses: "Funciona pra mim?"
- [ ] Addresses: "Quanto tempo leva?"
- [ ] Addresses: "E se nao funcionar?"
- [ ] Addresses: "Preciso de X primeiro?"
- [ ] Addresses: "Como e o suporte?"
- [ ] Format: question in bold
- [ ] Format: answer ends reinforcing benefit
- [ ] Position: before final CTA section
- [ ] Language: uses EXACT words customers use

---

## Phase 5: Estrutura de Valor

### 5.1 Stack de Valor [INFERRED]
- [ ] Total value displayed (e.g., R$15.000)
- [ ] Each component valued individually
- [ ] Shown BEFORE price reveal

### 5.2 Comparacao de Preco (Ancoragem Psicologica)
- [ ] Matrix includes: expensive alternative vs. complex alternative vs. slow alternative vs. your solution
- [ ] Uses formula: "Agencia: R$5k/mes | Consultor: R$15k | Fazer Sozinho: 2 anos | Voce: R$997"
- [ ] Position: immediately before or after value stack
- [ ] Function: makes your price seem "obvious" and "cheap"

### 5.3 Garantia Destacada
- [ ] Structure: multiple guarantee (Results + Satisfaction + Cancellation)
- [ ] Design: highlighted box
- [ ] Design: seal/badge
- [ ] Design: contrasting color
- [ ] Text example: "30 dias, sem perguntas"
- [ ] Text example: "Resultados em X dias ou 2X seu dinheiro"
- [ ] Position: near final price and CTAs

**Guarantee Template Check:**
```
GARANTIA TRIPLA "VITORIA CERTA":
✓ Implemente por 21 dias. Sem resultados? Dinheiro de volta.
✓ Nao ficar satisfeito por qualquer motivo? Dinheiro de volta.
✓ Aplicar tudo e nao ter ROI positivo? Devolvemos 2X seu investimento.
```
- [ ] Guarantee is specific (not generic "satisfacao garantida")
- [ ] Guarantee is conditional
- [ ] Guarantee may even be "unfair" to seller

## Phase 6: Autoridade e Confianca

### 6.1 Sobre o Criador/Autoridade
- [ ] Professional photo included
- [ ] 2-3 quantifiable credentials listed
- [ ] 1 vulnerability included (humanizing)
- [ ] Script follows: failure story → discovery → mission to help
- [ ] Position: after presenting solution, before showing price
- [ ] Tone: human, NOT "perfect guru"

### 6.2 Cases e Depoimentos Estrategicos
- [ ] Quantity: 3-5 specific cases
- [ ] All have numerical results
- [ ] Format: mix of text, video, before/after
- [ ] Distribution: spread throughout page (not all together)
- [ ] Testimonials are from avatars IDENTICAL to prospect

### 6.3 Logos e Credenciais
- [ ] Media appearances/mentions included
- [ ] Certifications displayed
- [ ] Client logos (if B2B)
- [ ] Impressive numbers: X clients, Y results
- [ ] Design: discreet but visible (not ostentatious)

## Phase 7: Conversao e Checkout

### 7.1 Multiplos CTAs Estrategicos
- [ ] Frequency: one CTA every 2-3 sections
- [ ] Variation: same CTA with slightly different text
- [ ] Position: above fold ✓
- [ ] Position: after benefits ✓
- [ ] Position: after value ✓
- [ ] Position: after guarantee ✓
- [ ] Position: final ✓
- [ ] Design: ALL identical in color/format (consistency)

### 7.2 Barra de Progresso (Checkout)
- [ ] Structure: 3 steps maximum (Info → Pagamento → Confirmacao)
- [ ] Design: clear visual of current progress
- [ ] Function: reduces anxiety and abandonment
- [ ] Impact: reduces abandonment 18-30%

### 7.3 Selos de Seguranca
- [ ] SSL indicator visible
- [ ] Credit card brand logos
- [ ] "Compra Segura" badge
- [ ] Privacy statement
- [ ] Position: in checkout area
- [ ] Position: near main CTAs
- [ ] Design: discreet but visible
- [ ] Function: eliminates final transaction fear

## 5 Fatal Errors Check

Before launch, verify NONE of these errors are present:

### Erro Fatal #1: Vender Recursos em Vez de Resultados
- [ ] NOT listing: "10 modulos + 50 templates + grupo no Facebook"
- [ ] Every feature has "...para que voce possa [RESULTADO EMOCIONAL]"
- [ ] Copy sounds like benefits, not ingredients

### Erro Fatal #2: Multiplas Ofertas na Mesma Pagina
- [ ] Only ONE offer on the page
- [ ] NOT showing: "Curso basico R$297 OU avancado R$997 OU consultoria R$5000"
- [ ] One page = one offer = one decision
- [ ] Remember: "Confused mind says no. Excited mind says yes."

### Erro Fatal #3: Garantia Fraca ou Inexistente
- [ ] Guarantee is NOT just "Satisfacao garantida"
- [ ] Guarantee exists (not missing)
- [ ] Guarantee is specific
- [ ] Guarantee is conditional
- [ ] Guarantee may be "unfair" to seller

### Erro Fatal #4: Preco Sem Contexto
- [ ] Price is NOT shown alone ("Investimento: R$997" - isolated)
- [ ] Value shown BEFORE price
- [ ] Comparisons present
- [ ] Guarantee accompanies price
- [ ] Urgency follows price

### Erro Fatal #5: Copy Generico de Nicho
- [ ] NOT using: "Transforme sua vida"
- [ ] NOT using: "Alcance seus sonhos"
- [ ] NOT using: "Metodo comprovado"
- [ ] Copy uses EXACT language avatar uses
- [ ] Page sounds like someone FROM the niche wrote it

## 48-Hour Validation Protocol

### Day 1: Auditoria Tecnica (2 horas)
- [ ] Teste em 3 dispositivos diferentes (smartphone, tablet, desktop)
- [ ] Carregamento < 3 segundos em conexao movel real
- [ ] Todos os links funcionando
- [ ] Formularios enviando corretamente
- [ ] Checkout processando pagamentos teste

### Day 1: Auditoria de Copy (2 horas)
- [ ] Teste dos 5 segundos com 5 pessoas
- [ ] Headline passivel de ser lida em voz alta naturalmente
- [ ] Cada secao responde uma objecao especifica
- [ ] CTA orientado a beneficio, nao acao
- [ ] Zero jargao tecnico desnecessario

### Day 2: Auditoria de Conversao (2 horas)
- [ ] Heatmap instalado e funcionando
- [ ] Eventos de conversao configurados
- [ ] Exit-intent popup testado
- [ ] Pixels de retargeting ativos
- [ ] Metricas de baseline definidas

### Day 2: Teste de Objecoes (2 horas)
- [ ] Mostrar pagina para 3 pessoas do avatar
- [ ] Perguntar: "Por que voce NAO compraria isso?"
- [ ] Documentar cada objecao
- [ ] Verificar se pagina responde cada uma
- [ ] Ajustar copy conforme necessario

## Troubleshooting Quick Reference

### Low Conversion (<3% cold)
- [ ] Check headline structure
- [ ] Check for 5 Fatal Errors
- [ ] Review heatmap for drop-off
- [ ] Verify social proof visible
- [ ] Audit guarantee strength

### High Scroll, Low Conversion
- [ ] Check CTA visibility
- [ ] Review price context
- [ ] List unanswered objections
- [ ] Audit checkout flow

### High Checkout Abandonment (>25%)
- [ ] Count checkout steps (should be 3 max)
- [ ] Check security seals
- [ ] Review price display
- [ ] Check for progress bar

### Low Mobile Conversion
- [ ] Run PageSpeed test
- [ ] Check button sizes (>44px)
- [ ] Count form fields
- [ ] Check text size (>16px)
