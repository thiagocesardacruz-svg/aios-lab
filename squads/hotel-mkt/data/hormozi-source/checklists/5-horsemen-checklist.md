# 5 Horsemen Implementation Checklist

> Checklist de implementação para `implement-5-horsemen.md` (HZ_RET_002)
> Source: $100M Retention Playbook - The 5 Horsemen of Retention

---

## Pre-Implementation

```yaml
pre_implementation:
  baseline_captured:
    - [ ] Churn rate atual documentado
    - [ ] Número total de clientes ativos
    - [ ] Capacidade da equipe avaliada
    - [ ] Sistema de tracking existente mapeado
```

---

## Horseman 1: Track Attendance

```yaml
horseman_1:
  system_setup:
    - [ ] Métrica de engajamento definida (logins, visitas, uso)
    - [ ] Baseline de "normal" estabelecido
    - [ ] Threshold de alerta definido (ex: -33%)
    - [ ] Sistema de tracking configurado
    - [ ] Alertas automáticos para equipe

  response_protocol:
    - [ ] Quem recebe alertas definido
    - [ ] Tempo de resposta definido (ex: 24h)
    - [ ] Script de outreach documentado
    - [ ] Escalation path claro
```

---

## Horseman 2: Reach Out 2x/Week

```yaml
horseman_2:
  cadence_setup:
    - [ ] Frequência mínima de 2x/semana estabelecida
    - [ ] Framework ACA documentado (Acknowledge, Compliment, Ask)
    - [ ] Canais de comunicação definidos
    - [ ] Schedule semanal criado

  team_training:
    - [ ] Equipe treinada no framework ACA
    - [ ] Exemplos de mensagens documentados
    - [ ] Tracking de outreachs implementado
```

---

## Horseman 3: Handwritten Cards

```yaml
horseman_3:
  program_setup:
    - [ ] Triggers definidos (signup, 3/6/12 meses, birthday, achievements)
    - [ ] Templates de mensagens para cada trigger
    - [ ] Supplies disponíveis (cards, stamps, pens)
    - [ ] Owner do programa atribuído

  execution:
    - [ ] Calendário de envios criado
    - [ ] Sistema de tracking de envios
    - [ ] Budget aprovado
```

---

## Horseman 4: Member Events

```yaml
horseman_4:
  calendar_setup:
    - [ ] Calendário de 6 meses criado
    - [ ] Eventos a cada 21/42/63 dias agendados
    - [ ] Tipos de eventos definidos (casual, learning, social, milestone)
    - [ ] Formato de convites definido (handwritten notes)

  logistics:
    - [ ] Venues/plataformas definidas
    - [ ] Guest policy clara (bring a friend)
    - [ ] Responsável por cada evento
```

---

## Horseman 5: Exit Interviews

```yaml
horseman_5:
  process_documented:
    - [ ] Expectativa setada no onboarding
    - [ ] Trigger de cancelamento configurado
    - [ ] Script de abertura documentado
    - [ ] 3 opções de save definidas (Redo, Upsell, Pause)

  team_preparation:
    - [ ] Equipe treinada nas técnicas
    - [ ] Resell protocol documentado
    - [ ] Tracking de saves implementado

  high_volume_alternative:
    - [ ] Cancellation video criado (se aplicável)
    - [ ] Video inclui: why started, value recap, what they lose
```

---

## Timeline Expectations Set

```yaml
timeline_communication:
  team_aligned:
    - [ ] Mês 1: Churn SOBE 50% - equipe preparada
    - [ ] Mês 2: Churn DESCE 50% - expectativa documentada
    - [ ] Mês 3: Estabilização - meta definida
    - [ ] Stakeholders informados do padrão 10%→15%→7%→3%
```

---

## Metrics Dashboard

```yaml
metrics_tracking:
  implemented:
    - [ ] Churn rate mensal
    - [ ] Save rate (exit interviews)
    - [ ] Outreach completion rate
    - [ ] Event attendance
    - [ ] Card send rate
    - [ ] Engagement alerts triggered
```

---

## Quality Gate

```yaml
quality_gate:
  all_horsemen:
    - [ ] Horseman 1: Tracking system LIVE
    - [ ] Horseman 2: Weekly outreach SCHEDULED
    - [ ] Horseman 3: Card program ACTIVE
    - [ ] Horseman 4: First event SCHEDULED
    - [ ] Horseman 5: Exit interview process DOCUMENTED

  owners_assigned:
    - [ ] Cada horseman tem owner claro
    - [ ] Métricas de sucesso por horseman definidas
```

## Scoring

| Category | Items | Points Each | Max |
|----------|-------|-------------|-----|
| Critical — Horseman Systems (tracking, outreach, cards, events, exits) | 20 | 3 | 60 |
| Standard — Pre-Implementation & Timeline Communication | 8 | 2 | 16 |
| Bonus — Metrics Dashboard & Quality Gate Owners | 13 | 1 | 13 |
| **TOTAL** | **41** | | **89** |

### Thresholds
- **PASS**: >= 72 points (80%)
- **CONDITIONAL**: >= 54 points (60%)
- **FAIL**: < 54 points

### If FAIL
Review which Horsemen are missing entirely and implement them sequentially starting with Horseman 1 (Track Attendance) as the foundation -- without tracking, no other retention tactic can be measured or improved.
