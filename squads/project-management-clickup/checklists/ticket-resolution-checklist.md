# Ticket Resolution Checklist

Use este checklist para garantir qualidade na resolucao de tickets de suporte.

## Metadata
```yaml
id: ticket-resolution-checklist
name: Checklist de Resolucao de Tickets
version: 1.0.0
executor: support-operations-specialist
```

---

## Triagem
- [ ] Ticket categorizado (acesso, pagamento, conteudo, tecnico)
- [ ] Prioridade definida (urgente, alta, media, baixa)
- [ ] SLA calculado baseado na prioridade
- [ ] Cliente identificado (produto, plano, historico)
- [ ] Informacoes iniciais suficientes para diagnostico

## Diagnostico
- [ ] Problema reproduzido ou compreendido
- [ ] Causa raiz identificada
- [ ] Solucao disponivel (KB, script, manual)
- [ ] Verificado se e problema recorrente
- [ ] Escalacao necessaria? (Se sim, para quem?)

## Resolucao
- [ ] Solucao aplicada ou orientacao fornecida
- [ ] Resposta clara e empatetica
- [ ] Tom de voz profissional e amigavel
- [ ] Links de referencia incluidos (se aplicavel)
- [ ] Screenshot/video explicativo (se necessario)
- [ ] Template de resposta utilizado (quando aplicavel)

## Follow-up
- [ ] Cliente confirmou resolucao?
- [ ] Satisfacao verificada
- [ ] Ticket atualizado com resolucao detalhada
- [ ] Tempo de resolucao dentro do SLA?
- [ ] Notas internas adicionadas

## Pos-Resolucao
- [ ] KB precisa de atualizacao? (Se sim, atualizar)
- [ ] Automacao pode prevenir reincidencia?
- [ ] Feedback registrado para melhoria do produto
- [ ] Metricas de atendimento atualizadas
- [ ] Ticket movido para "Resolvido" no ClickUp
