# Board Principles

> Princípios que governam o funcionamento do Advisory Board

## Core Principles

### 1. Advisory Only
O Board **nunca executa**. Apenas aconselha, questiona, e produz Decision Memos.

### 2. Multi-Perspective
Decisões importantes devem passar por múltiplos advisors com perspectivas complementares.

### 3. Conflict is Good
Desacordos entre advisors são features, não bugs. Documentar conflitos ajuda na decisão.

### 4. Output is Memo
Todo workflow produz um Decision Memo estruturado que pode virar OS via `/ops/new-task`.

### 5. Human Decides
O Director (humano) sempre faz a decisão final. Board informa, não decide.

## Decision Memo Standard

Todo memo deve conter:

1. **Context** - O que está sendo decidido
2. **Advisors Consulted** - Quem foi consultado
3. **Key Arguments** - Posição de cada advisor
4. **Conflicts** - Onde discordam
5. **Recommendation** - A recomendação
6. **Conditions** - O que precisa ser verdade para executar

---

## When to Consult Board

### Always Consult
- Mudanças de pricing/modelo de negócio
- Decisões irreversíveis (shutdown, pivot, contratação)
- Arquitetura de longo prazo
- Conflitos de princípios

### Consider Consulting
- Novas features significativas
- Parcerias estratégicas
- Expansão de mercado

### Don't Need Board
- Execução tática
- Bugs e fixes
- Tarefas operacionais diárias

## Anti-Patterns

### Don't
- Consultar Board para decisões táticas
- Ignorar conflitos entre advisors
- Pular direto para execução sem memo
- Usar Board como desculpa para não decidir

### Do
- Usar conflitos como input para decisão
- Documentar reasoning no memo
- Fazer decisão após receber memo
- Criar OS quando aprovar recomendação
